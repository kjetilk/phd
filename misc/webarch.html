Hypertext Transfer Protocol (HTTP)

  • Developed at Internet Engineering Task Force (IETF)
  • History
    • HTTP 0.9 informally defined in 1991.
    • HTTP 1.0 defined in RFC1945 in 1996.
    • HTTP 1.1 had several rounds of clarification:
      • RFC2068-2069 in 1997
      • RFC2616-2617 in 1999
      • RFC7230-7237 in 2014
    • HTTP/2 in 2015

Current HTTP specifications

  • RFC 7230 - HTTP/1.1: Message Syntax and Routing
  • RFC 7231 - HTTP/1.1: Semantics and Content
  • RFC 7232 - HTTP/1.1: Conditional Requests
  • RFC 7233 - HTTP/1.1: Range Requests
  • RFC 7234 - HTTP/1.1: Caching
  • RFC 7235 - HTTP/1.1: Authentication

HTTP Methods

  • GET, HEAD, POST, PUT, DELETE, CONNECT, OPTIONS, TRACE
  • Methods are also defined elsewhere (e.g. WebDAV)
  • Methods are classified as
    • Safe,
    • Idempotent,
    • Cachable.

HTTP Response codes

From the spec:

  • 1xx (Informational): The request was received, continuing process
  • 2xx (Successful): The request was successfully received, understood, and accepted
  • 3xx (Redirection): Further action needs to be taken in order to complete the request
  • 4xx (Client Error): The request contains bad syntax or cannot be fulfilled
  • 5xx (Server Error): The server failed to fulfill an apparently valid request

Some useful features

  • Content negotiation
  • Conditional requests
  • Cache treatment
  • Authentication
