Abstract

This essay takes the view of Semantic Web as a revolutionary technology, and initiates an epistemological discussion on how current empirical methodology is insufficient to gain knowledge on how to engineer it. It reviews some prominent directions in philosophy of science, and brings up the problem of evaluating approaches against a reality that is being created.

Introduction

>

Let us take as a starting point assume that science has a remarkable rate of progress because scientists use methodologies that are uniquely suited to enhance the common understanding of phenomena, and thereby create new knowledge. If this is true, then we have to understand the nature of knowledge to understand the methodology that is used. So, what does it mean to know something? That is one of the questions epistemologists concern themselves with.

Then, the question is if we can improve the rate of progress towards a Semantic Web by developing methodologies based on an improved understanding of epistemology.

The problem is not only epistemological, but also ontological: There isn't a clear, shared idea of what the Semantic Web is, and possibly, there shouldn't be. Clearly, the 2001 Sciam article has been an influencial vision, but my own vision predates that, and is somewhat different: I don't need the very intelligent agents for my vision, I just need a Semantic Web that is as ubiquitous as the Web is today, and brings down the cost of information integration. This envisioned Semantic Web does not exist as of today. Others may be content with the state of Linked Open Data or the use of large ontologies today, and therefore conclude that the Semantic Web exists. I would disagree, the number of JSON APIs with little semantics, the number of datasets based on comma-separated values, etc., overshadows the Semantic Web approaches completely. However, they do not provide the power to realize the applications envisioned in the revolutionary scenarios. It is my contention that the Semantic Web is revolutionary.

The rate of progress has not been what I expected when I first got interested in the Semantic Web around the time when the Semantic Web Roadmap was written. Can this lack of progress be explained in part by a lack of epistemological clarity, resulting in that the sizeable academic Semantic Web community has failed to provide scientific guidance to the development? And if so, why?
